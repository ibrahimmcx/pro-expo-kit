# 🚀 Pro-Expo-Kit (SDK 54)

> **A High-Performance, Reanimated-Free, Agentic Development System.**

Pro-Expo-Kit is a specialized environment for building React Native applications using **Expo SDK 54**. It is designed for maximum stability, visual excellence, and "Agentic Memory"—allowing AI assistants to learn and archive project-specific patterns.

---

## ✨ Key Features

- **SDK 54 Optimized**: Pre-configured for the latest stable mobile standards.
- **Reanimated-Free**: Focuses on the Native `Animated` API and `Moti` for ultra-stable performance without common runtime crashes.
- **Agentic Knowledge Base**: A built-in `.agent/` system that stores solutions, fixes, and architectural rules.
- **Kit Manager CLI**: A Python-based automation tool to manage memory and project health.

---

## 🛠️ Getting Started

### 1. Requirements
- Node.js & npm/yarn
- Python 3 (for Kit Manager)
- Expo Go or Development Build

### 2. Installation
```bash
# Clone the repository
git clone <https://github.com/ibrahimmcx/pro-expo-kit>
cd pro-expo-kit

# Install dependencies (when project is initialized)
npm install
```

### 3. Using the Kit Manager
The `scripts/kit.py` tool helps you maintain the kit's intelligence:

```bash
# View Kit status & "Memory"
python scripts/kit.py status

# Run SDK 54 compliance check
python scripts/kit.py check

# Archive a new technical solution
python scripts/kit.py archive "Fix Layout Bug" --content "Details here..."
```

---

## 🏗️ Architecture

- `.agent/`: The brain of the kit. Contains workflows, specialists, and knowledge items.
- `scripts/`: Python automation layer.
- `src/`: (To be created) Standard application source code.

---

## 📚 Principles

1. **Performance First**: No heavy reanimated overhead.
2. **Knowledge First**: Check `.agent/knowledge/` before reinventing a fix.
3. **Visual Excellence**: Every UI component must feel premium and state-of-the-art.
